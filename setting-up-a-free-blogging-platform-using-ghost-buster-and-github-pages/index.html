Setting up a free blogging platform using Ghost, Buster and Github Pages

19 February 2017 on guides

Overview

Ghost is blogging platform similar to WordPress. It lets you create and edit posts in markdown through a UI. Ghost provides hosting as well, but at $5/month for only 10,0000 page views.

Buster is a tool which can generate static content from a running Ghost instance and publishes that content to a Github pages repository. Using Buster you can generate content using Ghost and host it for free using Github Pages!

This is a step-by-step guide to setting all of this up. This blog was created using this exact same setup.

Create a working folder

$ mkdir ~/blog
$ cd ~/blog

Install dependencies

Install pip: 

$ sudo easy_install pip

Install Buster: 

$ sudo pip install buster

Download and install Ghost: 

$ wget https://github.com/TryGhost/Ghost/releases/download/0.11.4/Ghost-0.11.4.zip
$ unzip Ghost-0.11.4.zip -d ghost
$ cd ghost
$ npm install --production

Create Github Pages repository

Create a new repository with the following name:
https://github.com/<username>/<username>.github.io.git

The repository name must match this format exactly for Github to know to create a static site from this repository.

Configure Buster

$ buster setup --gh-repo="https://github.com/<username>/<username>.github.io.git"

Editing and publishing content

Start Ghost: 

$ npm start

Go to http://localhost:2368/ghost to edit content. The first time you run Ghost you will be asked to create a login account.

Once you are done creating or editing your posts, use Buster to generate the static content and push it to Github: 

$ buster generate
$ buster deploy
